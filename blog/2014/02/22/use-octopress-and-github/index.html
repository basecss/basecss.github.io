使用Octopress在Github搭建博客

Feb 22nd, 2014

虽然可选的程序有很多，但我喜欢 Octopress 的简洁，因而义无反顾的选择它。

早在去年 9 月折腾过一次使用 Octopress + Github 博客，希望用来记录一些信息。由于之前的误操作，把构建文件给删除了，这次推倒重来一次。下面简单记录一下折腾过程中的一些注意事项。

安装 Ruby 依赖

首先，由于 Octopress 是基于 Ruby 构建的，因此确保正确配置好相关的环境。我是在 Mac 环境中，系统自带的又 Ruby 支持，其版本为 1.8.7。而新版的 Octopress 需要 Ruby 1.9.3 及以上版本的支持，因此需要先升级一下 Ruby 版本。

搜索之下，决定使用 rvm (Ruby 多版本控制器)来升级 Ruby。在 Command line tool ok 的情况下，先安装好 rvm，使用如下命令即可。

curl -L get.rvm.io | bash -s stable
# 安装好 rvm 之后接下来安装 ruby
rvm install ruby 1.9.3

将 Octopress 克隆到本地安装

直接将 Octopress 程序 clone 到本地进行安装即可

git clone https://github.com/imathis/octopress.git dist

注意这里的 dist，自行指定在哪个目录安装 Octopress 即可。

clone 好 Octopress 之后。接下来安装好程序依赖：

# 先安装 bundler
gem install bundler

bundle install

最后再本地使用 rake 命令安装 Octopress：

rake install

至此，就安装好 Octopress 了。

配置 Octopress

Octopress 配置信息都在 __config.yml 文件中指定，修改其中相关配置即可。这里就不一一说明了，官方文档已经很清楚了。

写文章

在你的目标目录[博客目录]执行如下命令即可新建一篇文章：

rake new_post['fileName']

这个命令会在博客目录的 source/_post 中生成对应的 markdown 文件，使用 markdown 写好文章即可。

发布

在发布文章之前可以使用如下命令在本地预览一下：

rake preview

最后，托管到 github 即可。也可以选择使用 rake 命令配置你的仓库自动推送到仓库中：

# 配置 github 信息
rake setup_github_pages
# 执行这条命令根据提示配置好 github 信息，国内用户注意下 git url 协议，推荐使用 http 协议

# 最后使用如下命令构建
rake generate
# 部署到 github 上
rake deploy
